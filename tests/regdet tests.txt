1. Успешный сценарий

1.1 Открыть страницу. Ввести корректные имя, фамилию, LinkedIn, выбрать роль, загрузить валидное фото.
→ Кнопка Submit активна.
→ После нажатия Submit — переход на следующую страницу без ошибок.

2. Стартовое состояние

2.1 После открытия страницы поля имени, фамилии и LinkedIn пустые.
2.2 Радио-кнопки не выбраны.
2.3 Кнопка Submit заблокирована.

3. XFAIL — разблокировка при невалидных данных

3.1 Ввести невалидное имя и фамилию (1, q), выбрать роль.
→ Фактическое: кнопка активируется.
→ Ожидаемое: должна быть заблокирована.
→ @pytest.mark.xfail(reason="Submit активируется при любых непустых данных")

4. Валидация имени

4.1 Ввести невалидное имя (1). → Ошибка #name-error видна.
4.2 Очистить имя (clear_name()). → Поле пустое, ошибка исчезла.
4.3 Ввести корректное имя (John). → Ошибки нет.

5. Валидация фамилии

5.1 Ввести невалидную фамилию (q). → Ошибка #lastName-error видна.
5.2 Очистить фамилию (clear_lastname()). → Поле пустое, ошибка исчезла.
5.3 Ввести корректную фамилию (Smith). → Ошибки нет.

6. Валидация LinkedIn

6.1 Ввести некорректную ссылку (linkedin/test). → Ошибка #linkedin-error видна.
6.2 Очистить LinkedIn (clear_linkedin()). → Поле пустое, ошибка исчезла.
6.3 Ввести корректную ссылку (https://www.linkedin.com/in/testuser). → Ошибки нет.

7. Загрузка и удаление фото

7.1 Загрузить .jpg или .png нормального размера. → Ошибок нет.
7.2 Загрузить .txt. → Ошибка #format-size-error.
7.3 Загрузить файл, превышающий лимит. → Ошибка #format-size-error.
7.4 После успешной загрузки нажать Remove. → Фото удалено, кнопка загрузки снова активна.

8. Переключение роли

8.1 Выбрать “Startup”. → Активен только он.
8.2 Выбрать “Investor”. → Startup сбрасывается, активен Investor.

9. Logout

9.1 Кликнуть Logout. → Разлогин, редирект на страницу входа.
